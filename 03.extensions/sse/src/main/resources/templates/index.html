재연결 동작 순서:

3. 브라우저 EventSource: 연결 끊김 감지
4. 약 3초 후 자동 재연결 시도
5. Last-Event-ID 헤더로 마지막 이벤트 ID 전송
? SSE 프로토콜 핵심 개념

1 연결 수립 - HTTP 요청/응답

▶ 클라이언트 요청 (브라우저)
GET /sse/connect/user-001 HTTP/1.1
Accept: text/event-stream
Cache-Control: no-cache
Last-Event-ID: 5       ← 재연결 시만 포함

▶ 서버 응답 헤더
HTTP/1.1 200 OK
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive

2 정상 통신 - SSE 이벤트 구조

▶ SSE 이벤트 와이어 포맷
id: 7
event: message
data: {"type":"message","data":"Hello"}

← 빈 줄이 이벤트 종료를 나타냄

▶ 여러 줄 데이터
id: 8
event: message
data: line 1
data: line 2

3 재연결 - EventSource 자동 복구

▶ 재연결 흐름
1. 서버: emitter.complete() 호출
   → HTTP 스트림 종료

2. 브라우저 EventSource:
   - readyState = CONNECTING
   - 재연결 대기 (기본 3초)

3. 재연결 요청:
   GET /sse/connect/user-001
   Last-Event-ID: 7   ← 자동 포함

4. 서버: 마지막 ID 이후 이벤트 재전송
